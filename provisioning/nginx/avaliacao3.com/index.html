Proposta de Política de Segurança - Trabalho Final


André Fava, Antônio Cruz e Matheus Tamiozzo

Nesse documento estão descritas as regras da política de segurança que será implementada para o trabalho final da disciplina de Redes de Computadores II. Serão definidas três regras para cada serviço executado no box do trabalho final, sendo que três dessas regras serão consideradas regras diferenciadas.


Regras para o serviço de E-mail:

  • Bloqueio de domínios de e-mail (BlackList);
  • Bloqueios de tipos de anexo por extensão;
  • Bloqueio de envio de links no corpo do e-mail.

Regras para o serviço de FTP:

  • Bloqueio de upload tipo de arquivo por extensão (Ex: .exe);(Antônio)(Implementado)
  • Restrição de upload tamanho de arquivo (Ex: 50Kb);
  • Restrição de diretórios por IP ou Usuário (Ex: Pasta compartilhada para todos e pasta privada para usuário).

Regras para o serviço de Firewall:

  • Bloqueio de acesso de determinados IPs ao servidor de páginas (BlackList);
  • Bloqueio de aplicações peer-to-peer (Torrent);
  • Desabilitar saída de ssh do servidor.

Regras para o serviço de Proxy:

  • Controle de acesso por horário (Ex: Redes sociais liberadas apenas das 12:00 as 13:00);
  • Controle de largura de banda(Ex: Youtube com banda limitada);
  • Bloqueio de sites com expressões regulares (.io, .gov, .edu, .tech, etc.).

Regras para o serviço de servidor Samba:

  • Controle de acesso por usuários;
  • Controle de permissões em determinados diretórios (Ex: Diretório público read only e diretórios pessoais com escrita liberada);
  • Controle de Senha – Expira X dias ou Necessário X caracteres.

Das regras acima as seguintes foram definidas pelo grupo como “Regras Diferenciadas”

  • Proxy - Controle de largura de banda.
  • E-mail - Bloqueio de envio de links no corpo do e-mail.
  • Firewall - Bloqueio de aplicações a partir de um range de portas- Ex: peer-to-peer (Torrent) e portas.
